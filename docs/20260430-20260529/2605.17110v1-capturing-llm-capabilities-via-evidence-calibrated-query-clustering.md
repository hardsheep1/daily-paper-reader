## 摘要
查询聚类将查询组织成反映共享潜在能力需求的组，从而支持能力感知的LLM评估。现有的聚类方法主要依赖于语义分类或嵌入，由于表面级语义与实际模型性能之间的错位，往往无法捕捉这些潜在能力需求。我们提出ECC算法，该算法使用有限的后验模型比较来校准先验语义嵌入，以弥合表面级语义与潜在能力需求之间的差距。ECC通过参数化为Bradley-Terry模型的能力轮廓来表征每个聚类，并使用可训练的混合权重来适应具有混合能力需求的查询，联合学习一个灵活的、能力感知的聚类结构，支持查询特定的LLM能力推断。大量的定量和定性评估表明，ECC显著提高了LLM能力排序质量，分别比基于人工标注和基于嵌入的基线平均高出17.64和18.02个百分点，并在查询路由等下游任务中证明有效。

## Abstract
Query clustering organizes queries into groups that reflect shared latent capability demands, enabling capability-aware LLM evaluation. Existing clustering methods, which primarily rely on semantic taxonomies or embeddings, often fail to capture such latent capability requirements due to a misalignment between surface-level semantics and actual model performance. We propose ECC, an algorithm that calibrates prior semantic embeddings using limited posterior model comparisons to bridge the gap between surface-level semantics and latent capability requirements. ECC characterizes each cluster through a capability profile parameterized by a Bradley-Terry model and uses trainable mixture weights to accommodate queries with mixed capability demands, jointly learning a flexible, capability-aware clustering structure that supports query-specific inference of LLM capabilities. Extensive quantitative and qualitative evaluations demonstrate that ECC significantly improves LLM capability ranking quality, outperforming human-labeled and embedding-based baselines by an average of 17.64 and 18.02 percentage points, respectively, and proves effective in downstream tasks such as query routing.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有查询聚类方法（如基于语义分类或嵌入）仅利用表面语义信号，导致聚类与 LLM 的真实潜在能力需求严重错位。例如，同一语义类别（如“数学”）可能包含从死记硬背到多步推理等不同能力要求的查询；相反，具有相同能力需求但表面主题不同的查询却被分散到不同类别中。这种错位限制了能力感知评估的准确性和泛化性。
- **整体含义**：为了可靠地评估 LLM 在异构查询上的优势，需要一种新的聚类范式——利用有限的模型性能证据（后验比较）来校准先验语义，从而形成真正反映能力需求的聚类结构。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出 **ECC（Evidence-Calibrated Clustering）** 算法，实现“先评估，后聚类”。即在聚类过程中直接融入有限的成对模型比较（后验证据），而不是仅在聚类完成后才使用。
- **关键技术细节**：
  - **聚类表征**：每个簇 \(k\) 由两部分组成：
    - 语义质心 \(c_k\)（在嵌入空间中）；
    - 能力轮廓 \(\theta^{(k)}\)（参数化为 Bradley-Terry 模型），表示该簇上 M 个 LLM 的潜在强度。
  - **软责任机制**：每个查询 \(q\) 在所有簇上分配软责任 \(r_{qk}\)（非负且和为1），以处理混合能力需求。
  - **交替优化**：迭代更新系统参数 \(\Theta = \{\theta^{(k)}, c_k\}\) 和责任 \(\mathcal{P} = \{r_{qk}\}\)，目标函数为熵正则化的期望损失：
    \[
    \mathcal{L}(\mathcal{P}, \Theta) = \sum_{q,k} r_{qk} \ell(q, k; \Theta) - T \sum_q H(r_q)
    \]
    其中 \(\ell(q, k; \Theta) = \ell_{\text{comp}}(q; \theta^{(k)}) + \lambda \ell_{\text{emb}}(q; c_k)\)，\(\ell_{\text{comp}}\) 为基于负对数似然的 BT 损失，\(\ell_{\text{emb}}\) 为余弦相似度损失，\(\lambda\) 平衡后验与先验。
  - **闭式更新**：更新质心时使用归一化的责任加权平均；更新 BT 参数时最小化加权 BT 损失（凸优化）；更新责任时采用 Boltzmann 分布：
    \[
    r_{qk} \gets \frac{\exp(-\ell(q, k; \Theta)/T)}{\sum_j \exp(-\ell(q, j; \Theta)/T)}
    \]
  - **探针推断**：对于新查询，仅需一个成对比较（探针）即可校准责任，进而通过混合模型预测成对偏好：
    \[
    P(m_j \succ m_i \mid q) = \sum_k r_{qk}\,\sigma(\theta_j^{(k)} - \theta_i^{(k)})
    \]
  - **收敛性**：定理 2.1 证明目标函数单调非增且收敛到有限下界。

### 7. 优点

- **新颖性**：首次提出“先评估后聚类”范式，直接利用后验比较证据校准语义先验，解决了语义与能力错位的根本问题。
- **样本高效**：仅需少量成对比较（约 7 个）即可构建能力感知聚类；新查询仅需一个探针比较。
- **灵活性强**：软责任机制自然支持混合能力需求，无需硬分配。
- **无需额外训练**：ECC 完全在 CPU 上运行，收敛快，部署成本极低。
- **理论保证**：证明了目标函数的单调收敛性。
- **广泛验证**：在三大基准、多种设置和两个下游任务上均取得一致改进。

### 8. 不足与局限

- **实验覆盖**：仅使用文本查询，未评估多模态输入。但方法本身不限定嵌入模态，理论上可扩展。
- **缺乏上下文**：聚类仅依赖单条查询文本，未考虑对话历史或交互上下文。
- **评判器一致性**：虽然每个基准使用固定 LLM 评判器，但不同基准之间的评判器不同，可能引入噪声。作者在消融实验中检验了不同评判器的鲁棒性，证明 ECC 优势稳定。
- **探针成本**：虽然一次比较成本低，但需要额外生成两个模型响应。对于极大规模场景，仍可能增加延迟。作者建议可用轻量模型获取探针。
- **超参数敏感性**：\(\lambda\) 和 \(T\) 需调优，但实验表明合理范围（\(\lambda\approx3\)，\(T\approx0.1\)）性能稳定。
- **偏差风险**：聚类可能隐含地放大训练数据中的偏差，但论文未详细分析公平性。
