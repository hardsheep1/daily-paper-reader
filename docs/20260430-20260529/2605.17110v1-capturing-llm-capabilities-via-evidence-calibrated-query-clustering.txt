Abstract 

Query clustering organizes queries into groups that reflect shared latent capability demands, enabling capability-aware LLM evaluation. Existing clustering methods, which primarily rely on semantic taxonomies or embeddings, often fail to capture such latent capability requirements due to a misalignment between surface-level semantics and actual model performance. We propose ECC , an algorithm that calibrates prior semantic embeddings using limited posterior model comparisons to bridge the gap between surface-level semantics and latent capability require-ments. ECC characterizes each cluster through a capability profile parameterized by a Bradley-Terry model and uses trainable mixture weights to accommodate queries with mixed capability demands, jointly learning a flexible, capability-aware clustering structure that supports query-specific inference of LLM capabilities. Extensive quantitative and qualitative evaluations demonstrate that ECC signifi-cantly improves LLM capability ranking quality, outperforming human-labeled and embedding-based baselines by an average of 17.64 and 18.02 percentage points, respectively, and proves effective in downstream tasks such as query routing. 

1 Introduction 

Large Language Models trained on extensive corpora exhibit broad domain knowledge and diverse capabilities, as evaluated by standard benchmarks [ 1, 42 ]. Since different LLMs excel in different tasks (i.e. query-dependent ), accurately assessing their strengths is essential for reliable deployment in domain-specific and personalized settings [ 12 , 32 ]. However, coarsely curated datasets, or groups of queries, often yield noisy evidence that is insufficient for characterizing latent LLM capabilities, especially when queries within a group differ widely in their capability demands. This motivates clustering queries into subsets that share similar underlying capability demands, each corresponding to a distinct capability profile, which can enable a capability-aware LLM evaluation [48, 15]. Standard query “clustering” methods typically rely either on static and heuristic human-labeled semantic taxonomies, such as “Mathematics” or “Chemistry” [ 54 , 13 ], or on unsupervised grouping based on semantic embeddings [ 23 , 29 ]. While intuitive, such surface-level semantic clusters often fail to capture the true latent capability requirements of queries (Figure 1): (i) A single label (e.g., “Mathematics”) may encompass queries requiring vastly different levels of underlying capabilities, from rote recall to multi-step deductive derivation, or even a combination of distinct capability requirements. (ii) Conversely, queries requiring the same underlying capabilities, such as multi-step deductive derivation, may be scattered across different semantic clusters due to superficial topical differences. This misalignment with the true capability distributions can limit, or even degrade, the generalizability of model capability estimates to unseen queries (Figure 2), highlighting the insufficiency of prior semantic signals. Although recent works leverage “LLM-as-a-judge” [ 55 ] to extract explicit skill labels from model responses and build capability structures [ 30 , 44 ], the resulting

Furthermore, these approaches rely on exhaustively evaluating each query across numerous models to determine cluster assignments, which incurs substantial computational cost with LLM-based judges. These fundamental limitations motivate us to propose Evidence-Calibrated Clustering (ECC ), a novel algorithm that efficiently learns underlying capability-aware cluster structure by directly incorporating evidence of model performance, which we term posterior evidence , into the clustering process. Rather than relying solely on prior semantic embeddings, ECC incorporates limited pairwise model comparisons over query responses as sample-efficient signals of LLM capabilities, characterizing each cluster with a capability profile parameterized by a Bradley-Terry model [ 5 ] optimized over these comparisons. To accommodate queries requiring mixed capabilities, ECC introduces trainable soft responsibilities to weight the contribution of query embeddings and associated model comparisons across clusters, quantifying how strongly each query aligns with the capabilities represented by each cluster. To reconcile prior semantics with posterior evidence, ECC jointly optimizes cluster centroids, capability profiles, and soft responsibilities through alternating updates. For unseen queries, ECC employs probe-informed mixture inference , which leverages a single probe signal (i.e., a single pairwise model comparison) alongside the query embedding to derive a query-specific, capability-aware ranking over LLMs (see Section 2). We validate the effectiveness of ECC through extensive quantitative experiments across diverse benchmarks, demonstrating average improvements in ranking quality on unseen queries of 17.64 

percentage points (pp) over human-labeled clustering and 18.02 pp over embedding-only cluster-ing. We analyze the clusterings qualitatively to confirm that incorporating our posterior evidence yields interpretable capability-aware clusters, separating semantically similar queries with divergent capability needs and merging those with aligned needs. Finally, we apply ECC to two representative downstream tasks: (1) guided optimal query routing, where it improves the response quality of routed queries by 16.6% on average over embedding-only clustering, and (2) sample-efficient new model ranking, where it achieves 16.3 pp higher ranking quality under a limited comparison budget. In summary, our contributions are: 1. We propose ECC , a novel query clustering algorithm that integrates posterior model comparisons with prior embeddings to overcome the misalignment between surface-level semantics and latent capability requirements. 2. By characterizing each cluster with a capability profile parameterized by a Bradley-Terry model, 

ECC enables sample-efficient estimation of relative model strengths from limited pairwise com-parisons. 3. ECC captures the mixed capability requirements of queries by introducing trainable responsibili-ties, yielding a flexible, capability-aware clustering structure that enables query-specific inference of LLM capabilities. 4. Extensive quantitative and qualitative evaluations show that ECC produces more capability-coherent clusters and significantly improves capability-aware ranking quality, outperforming baselines in two downstream tasks. 

1.1 Related Work Capability-Aware Clustering for LLM Evaluation. Recent works move beyond human-defined taxonomies [ 45 , 52 , 24 , 40 , 3] by inducing latent capability structure from evaluation data for finer-grained LLM evaluation [ 44 , 30 , 49 , 26 , 31 ]. For example, SkillVerse [ 44 ] builds a hierarchical skill 2dendrogram using LLM-as-a-judge critiques, while other work extracts task attributes or knowledge components to guide clustering and improve performance prediction [ 30 , 49 ]. In contrast to these methods that derive fixed groupings from explicit attribute representations, ECC learns soft capability-aligned clusters that support query-conditional inference over LLM capabilities. 

Sample-Efficient LLM Evaluation. Full-benchmark evaluation is costly, as it requires scoring all models over thousands of items [ 22 , 7 ]. Recent work reduces this cost via comparison-based ranking and informative item selection [ 7, 38 , 22 , 20 , 4, 39 ]. For example, Chatbot Arena [ 7] fits Bradley-Terry models [ 5 ] to pairwise LLM comparisons to produce scalable global leaderboards, while ATLAS [ 22 ] applies IRT-based adaptive testing to estimate global model ability with far fewer benchmark questions. These methods primarily target a global leaderboard shared across queries, whereas ECC infers a query-specific mixture ranking over LLMs, enabling fine-grained adaptation across heterogeneous queries. See extended related work in Section E. 

1.2 Bradley-Terry Model for LLM Ranking. 

The Bradley-Terry (BT) model is a widely used framework for estimating latent strength scores of items from pairwise comparison data [ 5 ]. In the context of LLM evaluation, it has recently been adopted to rank a set of language models from limited pairwise comparisons over their responses to given queries [ 7]. i, j ∈ [M ] refer to two LLMs mi and mj , and y ∈ { 0, 1} denotes the comparison outcome over their responses to query q, with y = 1 indicating that mj is preferred over mi, and y = 0 otherwise. 2 Methodology 

The main intuition behind Evidence-Calibrated Clustering (ECC ) stems from a key limitation in existing query clustering methods that only rely on prior semantic embeddings: posterior evidence with informative model capability signals, such as pairwise model comparisons, is only used after clustering. This creates a fundamental gap: capability evidence cannot influence cluster formation, resulting in semantically coherent clusters that may still be misaligned with latent capability demands. 

ECC instead evaluates first, then clusters , where it first collects comparison evidence on unclustered queries and then uses it to calibrate and reshape the embedding-based semantic structure. Concretely, given an input number of clusters K, for each query q ∈ Q , ECC jointly leverages its semantic embedding xq and its associated pairwise comparisons Nq to learn a flexible latent capability clustering that enables reliable query-specific capability inference of LLMs. 

System Parameters Θ and Responsibilities P. To achieve this, ECC characterizes each specific cluster k ∈ [K] using two complementary components: (i) a semantic centroid ck, capturing the prior semantic signals in the embedding space, and (ii) a capability profile θ(k) ∈ RM parameterizing a BT model that represents the posterior capability strengths of the M LLMs on queries associated with the cluster. Together, these define the overall system parameters : Θ = {θ(k), ck}Kk=1 .3Since a single query may involve multiple capability demands, ECC adopts soft assignments rather than hard clustering by introducing a set of responsibilities P = {rqk }. For each query q, the responsibilities satisfy rqk ≥ 0 and PKk=1 rqk = 1 , where rqk measures how strongly q aligns with the capability profile represented by cluster k. Together, these cluster-level profiles define a 

fine-grained capability structure : each cluster serves as a basic capability unit, and each query forms its own capability representation by softly combining these units through its responsibilities. 

Alternating Optimization. Probe-Informed Mixture Inference. Since ECC learns weighted mixture capability clusters, inferring LLM capabilities for a new query q first requires estimating its soft assignment rq across clusters. A naive approach is to infer rq using embeddings alone; however, semantic proximity does not reliably imply capability proximity. Furthermore, this embedding-only inference ignores the fact that the learned cluster structure is jointly shaped by both embeddings and comparison evidence during clustering, resulting in a signal mismatch between clustering and inference. To reconcile this mismatch in a lightweight manner, we introduce a single probe signal during inference: for each query, we randomly collect only one pairwise model comparison and use it to calibrate rq via the same entropy-regularized update as in Equations (3) and (5), with ℓcomp computed on the probe only. 3 Evaluation 

Benchmarks. We evaluate on three widely used multi-LLM benchmarks: i) SPROUT [ 37 ], ii) RouterBench [ 17 ], and iii) Open LLM Leaderboard v2 [ 10 ]. The queries in these benchmarks span diverse datasets, including MMLU [ 15 ], MMLU-Pro [ 48 ], and MATH [ 16 ]. For each benchmark, all model responses are scored by the same judge to ensure consistency. We split queries into clustering/test sets with varying clustering proportions from 20% to 90%, using 80% by default. For each query, we decompose the full set of response scores into pairwise comparisons between models, using non-tied comparisons by default and considering tie-aware variants in the ablation study. During clustering, we sample a varying number of model comparisons per query (from 2 to 24), with 7 used by default, and vary the number of clusters from 2 to 100, using 30 as the default. For each test query, unless otherwise specified, we hold out one comparison as a probe for responsibility inference and evaluate on the remaining comparisons. We use bge-base-en-v1.5 [ 51 ] for query embeddings, and fix λ = 3 and T = 0 .1 unless otherwise specified.

Inference Signals. Given the learned clusters, we consider three inference signals for estimating the LLM ranking for a new query: (i) Emb-infer , which only uses the embedding signal of the query; (ii) 

Probe-infer , which only uses a probe model comparison of the query; (iii) Emb+Probe-infer , which combines both signals. Emb+Probe-infer is used as the default unless stated otherwise. 

Metrics. We use two main metrics to evaluate different clustering methods: (1) Clustering per-formance. We evaluate the effectiveness of a capability clustering structure via its impact on 

capability-aware ranking quality over unseen queries. Specifically, we report the ranking quality gains , defined as the average reduction in per-query BT loss (Equation (2)) on pair-wise model comparisons from unseen queries, relative to a single global BT model trained without clustering. (2) Hard partition structure analysis. To evaluate the structural properties of the hard cluster partitions (obtained by assigning each query to the cluster with the largest responsibility), we report Kendall-τ

to measure the consistency among the BT profiles across clusters, as well as Adjusted Rand Index (ARI) and Normalized Mutual Information (NMI) to quantify the agreement between two different partitions. Additional experimental details are in Section A. 

3.1 Main Results 

We compare different clustering methods across three inference signals, evaluating their performance on unseen queries with diverse capability demands, and address the following questions. 

RQ1: How does ECC compare to human-labeled clustering? We evaluate on three datasets with complete human-defined taxonomies: (i) MMLU with 57 subject categories, (ii) MMLU-Pro with 14 domain labels, and (iii) MATH with 7 problem types. For human-labeled clustering, we directly assign each query to its benchmark-provided ground-truth category and compute capability-aware ranking using the BT model trained within that category; thus, no additional test-time probe is needed for assignment. For ECC and embedding-only clustering, we use Emb+Probe-infer to compute query-to-cluster responsibilities and induce mixture rankings. For fairness, we set the number of clusters for ECC and embedding-only clustering to match the taxonomy granularity of each dataset. RQ2: How does the choice of clustering signals affect capability-aware ranking quality? To evaluate the impact of clustering signals, we compare ECC , embedding-only, and comparison-only clustering across benchmarks. As shown in Figure 3, on SPROUT, ECC consistently achieves the best ranking quality across inference signals (additional results for other benchmarks are provided in Figure 7). Compared to embedding-only clustering, Table 1 shows that with K=30 , ECC improves ranking quality gains by an average of 15.40 pp under Probe-infer, and 18.02 pp under Emb+Probe-infer across benchmarks. Even when inference relies solely on embeddings (Emb-infer), ECC still outperforms embedding-only clustering. The results highlight the limitations of relying on prior embeddings alone in clustering and the advantages of incorporating posterior comparison evidence during clustering. Notably, comparison-only clustering performs well only when probe signals are available, but degrades significantly under Emb-infer. This sensitivity reveals a signal mismatch: clusters learned purely from comparison evidence do not provide a reliable capability inference when the inference signal relies on embeddings alone. By integrating embeddings with comparison evidence, ECC produces a capability-aligned clustering structure that remains robust across inference signals, and outperforms comparison-only clustering under full-signal inference. These results suggest that embeddings serve as a complementary semantic prior to comparison evidence, enabling more accurate and robust alignment with diverse query capability demands. 

RQ3: How does the choice of inference signals affect capability-aware ranking quality? Holding the clustering method fixed and varying the inference signal, Figure 3 shows that incorporating even a single probe comparison at inference time substantially improves capability-aware ranking quality. Across methods, the strongest performance is achieved by combining the probe with embeddings (Emb+Probe-infer), which consistently yields higher ranking quality gains than using embeddings alone or the probe alone. Table 1 shows that for ECC , Emb+Probe-infer achieves the largest ranking quality gains, improving over Emb-infer by 19.06 pp and over Probe-infer by 5.64 pp. This suggests that using both embeddings and a probe comparison yields a stronger query-specific signal, leading to more accurate capability inference of LLMs. In contrast, embedding-only clustering exhibits only limited gains from adding the probe during inference, and in some cases, using the probe alone can even underperform Emb-infer. This confirms the signal mismatch: clusters formed purely from embeddings are not necessarily aligned with the comparison evidence signals during capability inference. As a result, probe evidence cannot be effectively exploited under embedding-only clustering, highlighting the importance of incorporating posterior comparison signals during clustering to enable robust and effective inference. 

RQ4: How do different clustering methods trade off within-cluster coherence and across-cluster separation in their hard partitions? We induce hard partitions by assigning each query to its highest-responsibility cluster and evaluate (i) within-cluster BT profile consistency via 5-fold Kendall-τ ,and (ii) cross-cluster separation via inter-cluster BT profile similarity and a cross-cluster ranking 7quality gap. Figure 4 shows that methods that incorporate comparison evidence during clustering (comparison-only clustering and ECC ) consistently produce more coherent and better-separated partitions across a wide range of K: they achieve higher within-cluster consistency, lower inter-cluster similarity, and larger cross-cluster quality gaps. In contrast, embedding-only clustering yields lower within-cluster consistency and weaker separation, suggesting that (i) clusters are less capability-consistent and tend to group queries with heterogeneous capability demands, resulting in unstable cluster-specific BT profiles, and (ii) queries requiring similar capabilities may be scattered across clusters, causing different clusters to induce similar BT profiles and providing limited capability-specific differentiation across LLMs. Overall, these results underscore the importance of incorporating posterior comparison evidence during clustering, which helps calibrate the obtained cluster structure to be more coherent within clusters and more separable across clusters, better reflecting the underlying capability demands across queries. Additional details and results for other benchmarks are provided in Section B.1. We further analyze how the number of clusters K impacts capability-aware ranking quality in Section B.2 and include additional comparisons with P2L in Section B.3. 

3.2 Cluster Overlap Analysis 

Figure 5 presents the cluster overlap matrices between hard partitions at K = 20 . Compared to embedding-only clustering, ECC exhibits concentrated overlaps: for most ECC clusters (rows), their queries predominantly fall into a small number of embedding-only clusters (columns). This indicates that ECC preserves a coarse embedding-only-induced scaffold rather than arbitrarily reshuffling queries (ARI= 0.108 , NMI= 0.221 ). ECC also aligns more closely with comparison-only clustering (ARI= 0.187 , NMI= 0.325 ), suggesting that the capability-relevant structure captured by posterior comparisons is retained in ECC . Embedding-only and comparison-only clusterings show minimal agreement (ARI= 0.021 , NMI= 0.054 ) and diffuse overlaps, indicating the misalignment between the two signals. ECC helps bridge these signals by retaining semantic structure from embeddings while using comparison evidence to refine clusters in a more capability-aware structure. We further examine the internal structure of the induced clusters before and after incorporating posterior model-comparison signals. A notable case appears in Figure 5: ECC_1 /ECC_10 and Emb_1 /Emb_10 form a prominent 2 × 2 block of four cells with large overlap counts. This suggests that the two ECC clusters largely draw queries from the two embedding clusters (and vice versa), but with a systematically different partitioning. To understand whether this repartitioning reflects semantic-topic shifts or deeper capability differences, we use GPT-5.2 [35] as a standardized summarization aid to generate 

human-verified post-hoc qualitative summaries for the overlap cells, including semantic topics and implied capability demands. These post-hoc summaries are used only for interpretation and are not used to define clusters or compute overlaps (full output in Section G). We first analyze within-embedding splits to understand why posterior model comparisons sep-arate queries that are close in embedding space. We fix the embedding cluster and compare the following intersection cells: ( ECC_1 ∩Emb_1 ) vs. ( ECC_10 ∩Emb_1 ), and ( ECC_1 ∩Emb_10 ) vs. (ECC_10 ∩Emb_10 ). Within Emb_1 , both cells fall into an advanced chemistry/biochemistry semantic region centered on mechanistic explanation and structure-property reasoning (e.g., “mechanism of the reaction between benzene and bromine”) (all example prompts are in Section H). However, posterior comparisons further separate this embedding cluster by different capability demands. Queries in (ECC_1 ∩Emb_1 ) tend to emphasize evidence-grounded inference from analytical or computational signals (e.g., “using infrared spectroscopy and proton NMR spectroscopy data interpretation”). In contrast, ( ECC_10 ∩Emb_1 ) focuses on constraint-aware design and procedure planning on top of mechanistic reasoning, introducing explicit optimization and safety/specificity constraints (e.g., “specifically targets the cell wall of Gram-negative bacteria”) and requesting a concrete simulation plan (e.g., “Use Molecular Dynamics simulations to investigate the interactions”). A similar pattern appears within Emb_10 (see Section B.4), further indicating that within a fixed embedding-defined semantic topic, posterior model comparisons can separate queries with distinct capability demands, yielding a more capability-aligned partition. We next analyze cross-embedding merges to understand why semantically different embedding clusters can be merged under posterior model comparisons. For a representative case, the two intersection cells within ECC_1 differ in semantics: ( ECC_1 ∩Emb_1 ) is framed in reaction/biochem-istry terminology (e.g., “concentration of the reactants affect the rate of the chemical reaction”), whereas ( ECC_1 ∩Emb_10 ) is framed in materials/device property control (e.g., “optical and electrical properties of liquid crystals be tuned”). However, posterior comparisons merge them into the same 8ECC_1 because both cells have the same capability demand: parameter-to-outcome causal modeling with simulation-supported prediction. This shared capability signature is evidenced by the consistent reliance on computational reasoning in both (e.g., “calculated using quantum chemistry methods”). A similar pattern holds for ECC_10 , see Section B.4 for details. These results suggest that embeddings primarily separate queries by surface semantic topic, while posterior comparisons can merge cells across embeddings when their capability demands align. In summary, ECC splits embedding-aligned queries when their latent capability demands diverge and merges semantically different queries sharing aligned capability demands. The resulting clusters are more capability coherent, making them better suited for LLM capability inference. Visualization of the learned clusters under different clustering methods is in Section B.5. 

3.3 Applications and Additional Studies Guided Optimal Query Routing. We consider an important downstream application of capability-aware ranking: optimal query routing , where the goal is to route each query to the model that maximizes response quality. For each test query, we infer a query-specific capability ranking over LLMs using a randomly sampled probe. The query is routed to the top-ranked model, and we record its response score. We report the average response quality over all test queries and compare against three baselines: (i) random routing, (ii) global routing to the top model from a single global BT ranking, and (iii) P2L [ 11 ], which learns an LLM predictor to directly output query-conditioned LLM leaderboard. As shown in Figure 6 (left), ECC achieves the highest average response quality on SPROUT across a wide range of K, outperforming all baselines. This demonstrates that ECC learns a more capability-relevant clustering structure, which enables better prediction of the best-performing model and leads to improved routing performance. Sample-Efficient New Model Ranking. 

We study another downstream task: insert-ing a new model into an existing ranking under a limited comparison budget. We evaluate sample efficiency by varying the number of comparisons involving the new model, with one comparison per selected query. Using only comparisons among ex-isting models, each clustering method first clusters queries and then uses a fixed bud-get of new-model comparisons to estimate the ranking of the new model. We evaluate on held-out new-model comparisons and report the ranking quality gains relative to the single global BT baseline trained with the same old-model comparisons and the same new-model budget. Figure 6 (right) on RouterBench shows that ECC achieves substantial ranking quality gains under small bud-gets, consistently outperforming embedding-only clustering, and is comparable to comparison-only clustering, demonstrating its effectiveness in leveraging limited comparison evidence for new-model ranking. 4 Conclusion and Limitations 

We present ECC , which calibrates prior semantic embeddings using limited posterior model com-parisons to bridge the gap between surface-level semantics and latent capability alignment. By characterizing clusters via capability profiles and introducing trainable responsibilities, ECC ac-commodates queries with mixed capability demands, yielding a flexible, capability-aware clustering structure for query-specific LLM capability inference. We validate ECC through extensive quanti-tative and qualitative evaluations, demonstrating significant improvements in ranking quality and confirming its effectiveness in downstream tasks. 9Our paper uses text-only queries and does not evaluate performance on multimodal inputs. However, 

ECC makes no assumptions about the embedding modality and can, in principle, accommodate multimodal query embeddings without changing its main clustering and inference procedures. We also cluster queries using only the standalone text of each query, without considering potential conversational context or historical context. Benchmarks. We use three widely used benchmarks: i) SPROUT [ 37 ], which covers 13 LLMs evaluated on a range of curated prompts; ii) RouterBench [ 17 ], which includes the responses from 11 different LLMs over diverse domain-specific queries; iii) Open LLM Leaderboard v2 [ 10 ], collects the queries and corresponding responses from 16 LLMs. These benchmarks span multiple datasets from diverse sources, including MMLU [ 15 ], MMLU-Pro [ 48 ], and MATH [ 16 ]. For each benchmark, all model responses are scored by the same powerful LLM-based evaluator to ensure consistency. We split the queries into clustering and evaluation sets with varying proportions (ranging from 20% to 90% for clustering), using a default split of 80% for clustering and 20% for evaluation. For each query, we decompose the full set of response scores into pairwise comparisons between models, using non-tied comparisons by default and considering tie-aware variants in the ablation study. During clustering, we sample a varying number of comparison pairs (from 2 to 24), with 7 pairs per query used by default. We vary the number of clusters from 2 to 100, using 30 as the default. For each test query, unless otherwise specified, we hold out one model comparison as a probe for responsibility inference and evaluate on the remaining comparisons. In the main setting, query embeddings are generated using bge-base-en-v1.5 [ 51 ]. We additionally evaluate SFR-Embedding-2_R [ 28 ] and 

text-embedding-3-large [ 34 ]. For both embedding and response generation, we use only the user query rather than the full input with the system prompt, avoiding contamination from additional system instructions and ensuring that embeddings reflect the query semantics. Unless otherwise specified, we fix λ as 3 and T as 0.1 for ECC . For convergence, we set the threshold for the average KL divergence between successive responsibilities to 10 −3, and the threshold for centroid stability to 

10 −4.

Baselines. We consider two baseline clustering methods: (i) Human-labeled clustering , which applies hard cluster assignments based on static, predefined taxonomies from human annotations; (ii) Embedding-only clustering , which uses only the semantic embeddings of queries for clustering and follows the same soft-assignment update style as ECC . Both baselines represent prior semantic clustering structures. We additionally consider a variant of ECC , Comparison-only (Comp-only) ,which follows the same optimization procedure but excludes the embedding signal by setting λ = 0 .For human-labeled clustering, we directly use the ground-truth labels as the cluster assignments for each query. Since each benchmark aggregates multiple datasets, some of which lack human annotations, we restrict our comparison to representative datasets that are internally consistent, drawn from the same source, and fully labeled. Specifically, we use three such datasets: (i) MMLU, (ii) MMLU-Pro, and (iii) MATH, all of which contain human-defined labels for every query. Inference Signals. Given the learned clusters, we consider three inference signals for estimating the LLM capability ranking for a new query: (i) Emb-infer , which only uses the embedding signal of the query; (ii) Probe-infer , which only uses a probe model comparison of the query; (iii) Emb+Probe-infer , which combines both signals. We use the Emb+Probe-infer signal as the default inference method, unless otherwise specified. Notably, all three inference settings follow the same posterior-weighted mixture ranking procedure described in Section 2, where they differ only in the information used to infer the query-specific responsibilities.

To further analyze cluster structure and separability, we compute two cross-cluster measures. First, for each K, we compute Kendall-τ between the BT profiles of every pair of hard clusters, and report the mean as the inter-cluster BT profile similarity with the shaded region showing the 25th–75th percentiles of these pairwise similarities. Second, for each target hard cluster, we compute a cross-cluster ranking quality gap by evaluating queries of that cluster under (i) BT models trained on all other clusters (and averaging the resulting ranking qualities) and (ii) the BT model trained on the same cluster. Overall, these results underscore the importance of incorporating posterior model-comparison evi-dence during clustering, which helps calibrate the obtained cluster structure to be more coherent within clusters and more separable across clusters, better reflecting the underlying capability distribution across queries. 

B.2 RQ5: How does increasing the number of clusters K affect capability-aware ranking quality? 

When varying K from 2 to 100, Figure 7 reveals a consistent scaling trend across benchmarks. Under Probe-infer and Emb+Probe-infer, ranking quality gains increase rapidly for small K and then saturate, with slight declines at very large K. In contrast, under Emb-infer, performance is relatively insensitive to K, indicating limited benefit from increasing cluster granularity when inference relies only on embeddings. Across all K and inference signals, ECC remains the best-performing method. However, extremely large K leads to a slight performance drop due to noisier cluster-specific BT estimates, as increasing K spreads the responsibilities mass across more clusters, reducing the effective comparisons supporting each cluster. Overall, a moderate number of clusters (K=20–50) suffices to capture the key capability variations among queries. 

B.3 Comparison with P2L 

We compare ECC with Prompt-to-Leaderboard (P2L) [ 11 ], a recent approach that trains an LLM-based predictor to directly output query-conditioned BT coefficients and induce per-query leader-boards from human-judged comparisons. We implement P2L following the same training objective and process as used in [ 11 ], using Qwen2.5-1.5B-Instruct [43 ] on each benchmark. As shown in Figure 9, ECC consistently achieves higher capability-aware ranking quality gains on unseen queries than P2L, across all three benchmarks and across different numbers of clusters. Notably, on both RouterBench and Leaderboard, P2L even underperforms the global BT baseline, resulting in negative ranking quality gains. These results suggest that directly learning an end-to-end query-to-ranking predictor can be sample-inefficient and sensitive to the amount and distribution of training data, especially when surface semantic signals are not well aligned with comparison-derived capability signals. Under limited supervision, P2L may overfit surface semantic patterns rather than recover the latent capability structure needed for query-specific capability inference. As a result, its predictions on unseen queries can be unstable and may even underperform the global BT baseline. In contrast, ECC 

explicitly learns reusable capability profiles from comparison evidence and performs query-specific inference through lightweight responsibility estimation, enabling more reliable rankings without training an additional LLM.

B.9 Guided Optimal Query Routing 

We consider an important downstream application of capability-aware ranking: optimal query routing ,where the goal is to route each query to the model that yields the highest response quality. For each clustering method, we construct query clusters with the clustering queries. During evaluation, for each test query, we randomly sample a single probe comparison (see Section D.3 for a discussion of probe cost) and combine it with the query embedding to infer a query-specific capability-aware ranking. The query is then routed to the top-ranked model under this inferred ranking, and we record its response score. We report the average response quality across all test queries and compare against three baselines: (i) Random routing, which routes each query to a randomly selected model, (ii) Global routing, which routes every query to the top model according to the single global BT ranking, and (iii) P2L [ 11 ], which learns a black-box LLM predictor to directly output query-conditioned LLM leaderboard. As shown in Figure 19, ECC consistently achieves the highest average response quality across all three benchmarks and a wide range of K, outperforming all baselines. Specifically, with K = 30 , ECC improves the response quality of routed queries by 16.6% on average over embedding-only clustering across three benchmarks. This demonstrates that ECC learns a more capability-relevant clustering structure that enables better prediction of the best-performing model for a given query, even with only a single probe comparison, leading to improved routing performance.

D Discussion 

D.1 Reducing Tied Comparisons 

In practice, tied comparisons can arise when two models perform similarly or when the evaluator is uncertain. Since ties provide weaker discriminative evidence than strict wins or losses and may reflect evaluator noise [ 55 ], including them can dilute capability-aware signals. Tie frequency can be reduced by using stronger evaluators or stricter judge prompts that reserve the tie option for cases where two responses are genuinely indistinguishable. 

D.2 Evaluation Judges 

Human judgments are an important source of preference signals in LLM evaluation [ 7]. In practice, however, they are often collected from multiple annotators with heterogeneous preferences and varying reliability, which can introduce noise and systematic bias into pairwise comparisons. Large-scale human evaluation is also substantially more expensive than automated evaluation. Therefore, for each benchmark, we use its designated LLM-based judge or fixed evaluation protocol to obtain consistent pairwise signals within that benchmark and avoid annotator-dependent variability. 

D.3 Evaluation Efficiency of ECC 

ECC is lightweight and does not require dense compute (e.g., GPU training [ 11 ]). Its optimiza-tion alternates between (i) updating cluster-specific BT capability profiles and (ii) updating soft responsibilities, both of which are efficient CPU operations. In our main setting with K = 30 , ECC 

29 converges within 32 iterations on SPROUT and RouterBench, and within 29 iterations on Open LLM Leaderboard. Furthermore, fitting BT models from limited pairwise comparisons is substantially cheaper than training additional neural components with gradient-based optimization. The main efficiency benefit of ECC lies in reducing evaluation cost. Whereas some existing methods rely on exhaustively judging responses from all models for every query, ECC only requires a small subset of model outputs and a limited number of pairwise comparisons per query. As shown in Figure 11, using around three comparisons per query already yields strong clustering performance. While ECC may require one probe signal for an unseen query (i.e., generating two model responses and performing one pairwise comparison), this overhead is O(1) per query and does not scale with the number of candidate LLMs ( M ). Moreover, since the probe signal is robust to the choice of probing models (Table 2), this probe can be obtained using lightweight models when evaluation cost is a concern. Thus, ECC provides an evidence-efficient approach to LLM capability evaluation compared with methods that require substantially more model outputs or pairwise judgments. 

E Extended Related Work 

Capability-Aware Clustering for LLM Evaluation. Recent work has moved beyond static, human-defined taxonomies and curated benchmark groupings [ 45 , 52 , 24 , 40 , 3] toward inducing latent capability structure from evaluation data for more fine-grained analysis [ 44 , 30 , 49 , 26 , 31 ]. Tian et al. [44] introduces SkillVerse, a diagnosis framework that utilizes an “LLM-as-a-Judge” to critique responses and organize them into a hierarchical skill dendrogram, revealing granular capabilities often missed by human labeling. Moayeri et al. [30] extracts latent task attributes, such as subtasks and domains, from model evaluation data, and uses them as signals to inform unsupervised clustering of prompts. On a related vein, Wei et al. [49] proposes KCluster, which leverages an LLM-induced question congruity metric to cluster assessment items into latent knowledge components, yielding clusters that predict student and model performance more accurately than expert-defined categories. However, these methods typically induce capability structure through explicit skill or attribute repre-sentations for diagnosis or performance prediction. In contrast, ECC does not rely on explicit skill extraction; instead, it directly integrates limited posterior evidence from pairwise model comparisons with prior semantic embeddings to learn soft capability-aligned clusters tailored to LLM ranking over diverse queries. Furthermore, whereas prior work often yields discrete partitions that support group-level analysis once the grouping is fixed, ECC learns soft capability-aligned clusters that support 

query-conditional inference over LLM capabilities, enabling stronger adaptation and generalization under limited comparison evidence. 

Sample-Efficient LLM Evaluation. Modern LLM evaluation is costly as a comprehensive assess-ment requires scoring all LLMs over thousands of benchmark records [ 22 , 7 ]. To reduce evaluation cost, recent work has developed sample-efficient methods based on adaptive item selection, bench-mark compression, and comparison-based ranking [ 22 , 20 , 7, 38 , 39 ]. For instance, Chatbot Arena [ 7]collects crowdsourced pairwise preferences and fits Bradley-Terry models [ 5 ] to produce scalable global leaderboards without exhaustively evaluating every model on every prompt. In parallel, Item Response Theory (IRT) provides a principled measurement framework for improving sample ef-ficiency [ 4]. Song et al. [39] proposed IRT-Router, which models latent model ability and query difficulty to support cost-effective multi-LLM routing, improving over semantic-routing baselines. ATLAS [ 22 ] uses IRT-based adaptive testing (e.g., Fisher-information-guided item selection) to esti-mate model ability accurately with far fewer benchmark questions, while Metabench [ 20 ] leverages redundancy in large-scale leaderboard data to distill sparse benchmarks that preserve score estimates. Furthermore, Arena-Lite [ 38 ] improves ranking reliability under limited budgets by organizing LLM-judge comparisons in a tournament structure, reducing the number of required comparisons. However, most prior work primarily targets sample-efficient estimation of global model scores or leaderboards, which is a single ranking shared across all queries. In contrast, we target capability-aware ranking under heterogeneous query demands: ECC learns latent capability clusters and performs per-query posterior inference to produce a query-specific mixture ranking over LLMs. This enables fine-grained adaptation across queries while remaining effective under limited comparison budgets. 

Query Routing . Recent work studies how to match heterogeneous queries to the most suitable model under quality-cost constraints. A common line of approaches invokes multiple models for each query, either by ensembling or reranking candidate responses [18, 46] or by cascading models from cheaper to more expensive ones until a stopping criterion is met [ 6, 2, 53 , 21 , 39 ]. For instance, LLM-30 Blender [ 18 ] combines and reranks responses from multiple LLMs to produce a final output, and Frugal-GPT [ 6] sequentially queries models in increasing capability to reduce cost while maintaining quality. Another line trains a router on historical or auxiliary data to estimate the performance and cost of each query and select the optimal LLM accordingly [ 33 , 37 , 17 , 8 , 25 , 50 , 36 , 9, 14 , 47 , 41 , 19 ]. For example, RouterLLM [ 33 ] trains a lightweight BERT model on preference-labeled data to improve selection accuracy, and CARROT [ 37 ] learns predictors of both quality and cost to support flexible trade-offs. RadialRouter [ 19 ] proposes a lightweight Transformer backbone, RadialFormer, to model query-LLM relationships for routing, while GraphRouter [ 9] uses a heterogeneous graph to capture interactions between query requirements and model capabilities. These routing methods typically learn a per-query selector, whereas ECC learns a capability-aware clustering structure and outputs a query-specific mixture ranking over models, with routing as a special case (selecting the top-ranked model) while enabling broader capability-aware evaluation tasks. One recent work [ 11 ]introduces Prompt-to-Leaderboard (P2L), which trains an LLM-based predictor on human-annotated pairwise comparisons to directly estimate query-conditioned BT coefficients, thereby inducing a per-query LLM leaderboard. In contrast, ECC differs fundamentally in both goal and methodology. P2L focuses on a specific application that learns an amortized black-box predictor for per-query leaderboard generation (e.g., for query routing), without modeling any interpretable latent capability structure across queries. ECC , on the other hand, aims to uncover a fundamental capability-aware clustering structure that reveals how queries relate based on underlying capability demands – a structure that is both interpretable and broadly applicable to multiple downstream tasks, including query routing and new model ranking. Furthermore, P2L requires costly model training, whereas 

ECC is training-free and efficient, and executable on commodity hardware such as CPUs. 

F Broader Impact 

This paper advances machine learning by proposing a clustering framework that enables more fine-grained, capability-aware LLM ranking across queries with diverse capability requirements. As a methodological contribution, it introduces limited direct societal risk. By providing more reliable LLM rankings, it can help users choose models that better match their needs, improving downstream application outcomes for a broader range of users.
