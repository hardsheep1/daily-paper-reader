## 摘要
最近的文本聚类方法使用大语言模型从语料库中提出聚类分类体系，然后将每个文本分配给它。这些流程本质上是程序化的：LLM调用的序列以及停止、合并和拆分聚类的规则预先在代码中固定，因此它们在跨不同结构的语料库时泛化能力差，并且不容易整合用户提供的约束，例如目标聚类数或聚类意图。我们提出了一种基于智能体的替代方案，其中协调LLM在每一步检查发现过程的状态，并派遣一组专业化智能体中的一个——提议者、合成者、审计者、调查者和批评者——使流程适应语料库，而不是执行固定的流程。在七个公开的文本聚类基准上，该方法实现了最先进的性能，在ARI上比先前最强的LLM基线高出多达32%。

## Abstract
Recent text-clustering methods use large language models to propose a cluster taxonomy from a corpus and then assign each text to it. These pipelines are fundamentally programmatic: the sequence of LLM calls and the rules for stopping, merging, and splitting clusters are fixed in code in advance, so they generalise poorly across corpora of different structure and cannot easily incorporate user-supplied constraints such as a target cluster count or a clustering intent. We propose an agentic alternative in which an orchestrator LLM inspects the state of the discovery process at each step and dispatches one of a small set of specialised agents - proposer, synthesizer, auditor, investigator, and critic - adapting the pipeline to the corpus rather than executing a fixed one. On seven public text-clustering benchmarks the method achieves state-of-the-art performance, beating the strongest prior LLM baseline by up to 32% in ARI.

## 1. 核心问题与整体含义

- **研究动机**：现有基于大语言模型的文本聚类方法采用**固定程序化流水线**（proposers、merge/split规则等代码中固定），导致在不同结构语料上泛化能力差，且难以融入用户约束（如目标聚类数、聚类意图）。

## 2. 方法论

### 核心思想
- 采用**多智能体框架**，由一个**协调器（Orchestrator）** 根据当前状态（如审计结果、批评反馈）动态调度以下智能体：
  - **提议者（Proposer）**：从语料采样生成候选聚类（多粒度并行）。
  - **合成者（Synthesizer）**：合并多组提案为统一分类体系，受k范围约束。
  - **审计者（Auditor）**：用未见文本验证分类体系的覆盖率和置信度。
  - **调查者（Investigator）**：针对具体问题（如是否合并/拆分）进行检索并给出建议。
  - **批评者（Critic）**：从六个维度（重叠、空白、粒度、边界、描述质量、k范围）评估分类体系。

### 关键技术细节
- **输入**：未标注语料 \(D\)、目标k范围 \([k_{\min}, k_{\max}]\)、可选的任务描述（聚类意图）。
- **提议阶段**：多个提议者并行，从不同粒度（平衡、细粒度、粗粒度）采样，使用**顺序不相交采样**（sequential disjoint sampling）保证样本不重叠。
- **合成阶段**：根据提案语义相似性和直接检查证据文本进行合并，保留仅出现在一个提案中的低置信度聚类。
- **审计/调查/批评循环**：审计者检查未被样本覆盖的文本；调查者通过TF-IDF检索或具体ID定位问题；批评者对分类体系进行多轴评估。
- **最终分类**：使用廉价LLM（GPT-5-mini）将每个文本分配到分类体系中的聚类或`none`，支持可选的提示优化（若有少量标注数据）。
- **流程非程序化**：协调器根据状态决定下一步派遣哪个智能体及如何处理其返回值，不是固定顺序。

## 6. 主要结论与发现

- **性能**：本文方法在**所有数据集上达到或最优**（CLINC150上LLM-embedding+k-means平局），多数数据集ARI领先第二方法**超过10%**（最高32%）。
- **粒度控制优势**：其他LLM分类法方法（TopicGPT、Huang & He）在无真实标签种子时无法控制粒度，产生过多或过少聚类；本文方法通过用户约束和动态调整实现有效控制。
- **消融结果**：
  - 去除“聚类意图”导致部分数据集性能明显下降（如CLINC150 ARI从0.39降至0.24）。
  - 去除后处理循环影响很小，说明主要贡献来自提议和合成阶段。
- **成本**：本文方法成本并非最高（低于TopicGPT和Huang & He），且质量领先。
