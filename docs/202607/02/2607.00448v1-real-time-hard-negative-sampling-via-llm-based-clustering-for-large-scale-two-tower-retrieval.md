## 摘要
双塔模型已广泛应用于大规模推荐系统，特别是在检索阶段。训练双塔模型的行业标准通常涉及批内和/或批外负采样。然而，这些方法通常产生模型能快速学习的简单负样本，未能充分挑战模型。为解决此问题，提出了一种新颖的自监督硬负采样技术，利用大语言模型（LLM）在模型训练期间从同一聚类生成硬负样本。通过利用LLM学习媒体表示，所提方法确保生成的负样本更具挑战性和信息量。该实时采样框架旨在无缝集成到生产模型中，能够以最小计算复杂度处理数十亿训练数据点。在公共数据集上的实验以及在大规模在线系统中的部署表明，所提负采样技术优于广泛使用的行业方法。此外，工业应用中的分析表明，该采样方法有助于打破推荐中固有的反馈循环，并显著减少流行度偏差。

## Abstract
The two-tower model has been widely used for large-scale recommendation systems, particularly in the retrieval stage. Industry standards for training two-tower models typically involve in-batch and/or out-of-batch negative sampling. However, these methods often produce easy negatives that models can quickly learn, failing to sufficiently challenge the model. To address this issue, a novel self-supervised hard negative sampling technique is proposed that leverages a large language model (LLM) to generate hard negatives from the same cluster during model training. By utilizing the LLM to learn media representations, the proposed approach ensures that the generated negatives are more challenging and informative. This real-time sampling framework is designed for seamless integration into production models, capable of handling billions of training data points with minimal computational complexity. Experiments on public datasets, along with deployment to a large-scale online system, demonstrate that the proposed negative sampling technique outperforms widely used industry methods. Furthermore, analysis in industrial applications reveals that this sampling method can help break inherent feedback loops in recommendations and significantly reduce popularity bias.

## 2. 论文提出的方法论

### 核心思想
- 利用**LLM构建多模态内容嵌入模型**，将媒体（文本、图像、视频）映射到语义丰富的向量空间，并基于该空间对全部物品进行聚类。
- 在双塔模型训练时，对每个正样本（user-item对），从其所属聚类中随机采样若干物品作为**硬负样本**（同一簇内的物品与正样本语义相似），替代或补充传统的随机负样本。

### 关键技术细节
1. **LLM聚类生成**（三阶段流程）：
   - 预训练：使用预训练LLM获得通用语言理解能力。
   - 多模态编码器：基于Transformer处理文本、图像、视频输入，输出固定大小的向量表示。
   - 微调：针对特定任务微调，使聚类更贴合用户兴趣和内容语义。
2. **GOOBS实时采样框架**（Global Out-of-Batch Sampling）：
   - 维护一个全局**OOB物品池**，每个物品按聚类ID通过哈希函数映射到特定槽位（slot）。
   - **更新引擎**：每个训练batch处理完毕后，将当前batch中的物品特征通过哈希写入池中对应聚类段。
   - **采样引擎**：根据当前batch中正样本的聚类ID，从相应聚类段中随机选取一个物品作为OOB负样本（替换或补充批内负样本）。
   - 预加载：训练初期从历史数据预填充物品池，确保高命中率。
3. **损失函数**：对每个正样本和其K个聚类负样本计算交叉熵损失（仅对正样本和负样本做softmax，而非全量物品）。

### 公式/算法流程（文字说明）
- 输入：训练batch包含B个物品ID及其聚类ID。
- 更新：对每个物品ID，计算 `cluster_id * segment_size + ID mod segment_size` 作为槽位索引，将该物品特征写入池中。
- 采样：对每个batch中的物品，根据其聚类ID随机选择对应段内一个槽位的已有物品作为负样本。
- 训练：正样本与采样出的OOB负样本组成mini-batch，计算softmax交叉熵损失。

## 3. 实验设计

### 公开数据集与场景

### Benchmark方法对比
- **Baseline**：in-batch负采样 + LogQ校正 + 假阴性掩码。
- **DNS**（Dynamic Negative Sampling）：利用当前模型打分最高的非正样本。
- **CBNS**（Cross-Batch Negative Sampling）：缓存最近batch的item嵌入作负样本。
- **ANCE**（Approximate Nearest Neighbor Negative Contrastive Estimation）：建立全局ANN索引，选取几何上最近的负样本。
- **GOOBS**（随机OOB）：从物品池中随机采样OOB负样本。
- **Cluster GOOBS**：从同一聚类中采样OOB负样本（随机与聚类的比例1:15或1:31）。

### 工业数据集
